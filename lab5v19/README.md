**2. Опис Реалізації**
Змодельовано предметну область "Пакування на складі" (Варіант 19) згідно вимог:
Структура та Зв'язки
Композиція: Реалізовано клас PackPlan (план пакування/бокс), який композиційно містить колекцію (List<BoxItem>) об'єктів BoxItem (предмет).Валідація: У конструкторах BoxItem та PackPlan додано валідацію вхідних даних (розмір $> 0$, місткість $> 0$).
**Generics-компонент**
Створено узагальнений клас PriorityQueue<T>, що використовує SortedSet<T> для підтримки черги з пріоритетом.Визначено допоміжний клас PrioritizedBoxItem, який реалізує IComparable<T>, щоб PriorityQueue міг сортувати предмети за їхньою цінністю (Value) у спадному порядку.
**Логіка та Обчислення**
 (LINQ)Реалізовано "жадібний" алгоритм пакування у Program.cs, який використовує PriorityQueue для вибору найцінніших предметів першими.Клас PackPlan використовує LINQ для обчислення ключових метрик:CurrentFillSize: Загальний розмір запакованих предметів (.Sum(item => item.Size)).RemainingSpace: Залишковий простір (MaxCapacity - CurrentFillSize).FillPercentage: Відсоток заповнюваності.У Program.cs також обчислюється сумарна та середня цінність запакованого вантажу.
 **Обробка Винятків**
 Реалізовано та продемонстровано обробку двох власних класів винятків:InvalidDataException: Кидається при спробі створити об'єкт BoxItem з некоректними даними (наприклад, негативний розмір).ItemTooLargeException: Кидається у методі PackPlan.AddItem, якщо розмір предмета перевищує максимальну місткість боксу.Усі виклики, що потенційно можуть спричинити винятки, обгорнуті у блоки try-catch у Program.cs.
