**Відповіді на запитання**

1. Що таке Generics? Які їхні переваги?
Generics (Узагальнені типи) — це функція мови C# та .NET, яка дозволяє створювати класи, методи та структури, що можуть працювати з даними будь-якого типу, при цьому зберігаючи типову безпеку та продуктивність. Тип даних (T) визначається користувачем під час створення екземпляра (наприклад, List<string>, Repository<PackPlan>).
Переваги Generics:
Типова безпека (Type Safety): Компілятор перевіряє сумісність типів на етапі компіляції, усуваючи необхідність у ручному перетворенні типів (casting) та запобігаючи помилкам часу виконання (InvalidCastException).
Продуктивність (Performance): Уникає операцій упаковування/розпаковування (boxing/unboxing), які необхідні при роботі з типом object (як у старих колекціях, наприклад, ArrayList). Це значно швидше, особливо для значущих типів (int, struct).
Повторне використання коду: Один узагальнений клас (наприклад, PriorityQueue<T>) може обслуговувати безліч різних типів даних.

2. Які основні відміності між  ArrayList i List <T> ?

ArrayList - це стара колекція, яка зберігає елементи як object. Приймає будь які типи, бо все зберігається object.Для значимих типів (int, double і тд) відбувається boxing-unboxing торхи повільінше.
List<T> - це узагальнена generic колекція, яка зберігає елементи конкретного типу Т. Має сильну типизацію, безпечніший і кращий по продуктивності, зручний з LINQ.

3. Чим відрізняється Dictonary<Tkey,Tvlaue> від List<T> ?

List<T> - це просто список елементів по індексу 0,1,2...
Dictionary - це набір пар ключ-значення, де ми звертаємось по ключу.

4. У чому перевега LINQ над класичними циклами ?

LINQ дозволяє записувати до колекцій коротко й зрозуміло в одному виразі замість купу циклів for та foreach.

5. Як працює ключове слово finally?

Блок finally - це частина кострукції try-catch....finally, яка виконується завжди, і коли виняток стався, і коли ні, навіть якщо всередині try був return.

6. Коли доцільно створювати власні класи винятків ?

Варто створбвати тоді, коли стандартні типи погано пояснюють суть помилки, і нам потрібен більш конкретний варіант.
